# SeedLink 조회 중심 API 성능 테스트 결과

## 테스트 목적

SeedLink의 조회 중심 API가 동시 사용자 증가 상황에서 안정적으로 응답하는지 확인했습니다.

이번 테스트는 마일스톤, 정산, 결제 상태를 변경하는 테스트가 아니라 조회 API 성능을 확인하는 테스트입니다.

## 테스트 환경

- 테스트 도구: k6
- 테스트 환경: 로컬 dev 환경
- 부하 발생기와 백엔드 서버: 동일 로컬 장비에서 실행
- 테스트 데이터: `performance/seed/performance-seed.sql`
- 기준 스크립트: `performance/k6/seedlink-local-read.js`

## 테스트 대상 API

- 아이디어 목록 조회
- 인기 아이디어 Top5 조회
- 펀딩 목록 조회
- 펀딩 상세 조회
- 아이디어 상세 조회
- 마일스톤 목록 조회
- 보고서 목록 조회
- 자금 사용 내역 조회
- 가상계좌 장부 조회

환불 생성, 환불 완료/실패 처리, 강제 환불, 선정산 신청, 보고서 승인/반려처럼 상태를 변경하는 API는 이번 테스트에 포함하지 않았습니다.

## 성능 기준

- 실패율: 1% 미만
- p95 응답 시간: 1초 미만
- p99 응답 시간: 3초 미만

## 결과 해석

조회 중심 API는 local, normal, target, scale 단계에서 모두 실패 없이 통과했습니다.

500명 동시 사용자 부하에서도 실패율 0%, p95 307.95ms, p99 373.6ms로 안정적인 응답을 보였습니다.

1,000명 동시 사용자 부하에서는 실패율 0%와 체크 성공률 100%를 유지했지만, p95 응답 시간이 1.01초로 기준인 1초를 약 0.01초 초과했습니다.

따라서 조회 중심 API는 로컬 환경 기준으로 500명 동시 사용자까지 안정적으로 처리 가능하며, 1,000명 부하에서도 요청 실패는 발생하지 않았으나 응답 시간 기준은 근소하게 초과했습니다.

## 특이사항

초기 테스트에는 `/actuator/health` 요청이 포함되어 있었으나, 로컬 환경에서 해당 엔드포인트가 정상 응답하지 않아 전체 실패율과 p95 응답 시간이 왜곡되었습니다.

서비스 기능 API가 아닌 서버 상태 확인용 엔드포인트이므로, 조회 API 성능 측정 대상에서 제외한 뒤 재측정했습니다.

## 한계

- 이번 테스트는 조회 중심 API만 대상으로 했습니다.
- 환불, 정산, 선정산, 보고서 승인/반려 등 상태 변경 API의 정합성은 별도 테스트가 필요합니다.
- 로컬 장비에서 서버와 부하 발생기를 함께 실행했기 때문에 500명 이상부터는 로컬 장비 성능의 영향을 받을 수 있습니다.
- Seed 데이터 규모가 실제 운영 데이터보다 작으므로, 대량 데이터 환경에서는 추가 검증이 필요합니다.

## 결론

조회 중심 API는 로컬 성능 테스트에서 500명 동시 사용자까지 안정적으로 통과했습니다.

1,000명 동시 사용자 테스트에서도 실패율은 0%였지만, p95 응답 시간이 기준을 근소하게 초과했습니다.

상태 변경이 포함되는 환불, 정산, 선정산, 마일스톤 승인/반려 흐름은 별도 정합성 테스트로 검증해야 합니다.
